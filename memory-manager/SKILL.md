# Memory Manager

分层记忆管理。核心原则：**越频繁调用的记忆越要精简，越细节的内容越要下沉。**

## 记忆架构

```
.memory/
├── hot.md        # 热记忆：每次会话必用的核心事实，严控 <600 tokens
├── context.md    # 温记忆：详细背景、决策历史、已知问题，按需读取
├── user.md       # 用户画像：跨项目/跨领域的稳定偏好，跨工作区复用
├── guidance.md   # 主动建议：给下次会话的待办与提醒，每次覆盖写入
├── log.md        # 会话日志：历次摘要，倒序追加
└── sop/          # 程序性记忆：可复用的操作流程（SOP 文档）
    └── [task].md
```

**层级选择规则**：

```
几乎每次都用 → hot.md（≤ 1-2 句话）
偶尔需要但忘了会出错 → context.md（详细记录）
是可重复执行的多步骤流程 → sop/[task].md
关于用户本人的偏好/风格 → user.md
给下次会话的优先提醒 → guidance.md
```

**无目录权限时**：合并为单文件 `memory.md`，用 `## Hot` / `## Context` 等标题分区。

## 操作模式

### 读取记忆（会话开始）

触发词：「看看记忆」「回顾上次进度」「我们上次做到哪了」

检查 `.memory/` 是否存在，存在则读取并简述：

```
📋 记忆恢复

上次建议：[guidance.md，若有]
遗留事项：[从 context.md 或 hot.md 中提取 pending 信息]
当前上下文：[hot.md 核心要点]
```

读取后不修改任何文件。

---

### 初始化（从零建立）

触发词：「初始化记忆」「帮我建立记忆系统」「这是新项目/工作」

**Step 1**：扫描工作目录，了解基本情况（有代码则读 README+配置文件+对工程进行扫描分析建立认知，无代码则分析项目已有文件并建立项目整体认知）。

**Step 2**：向用户提问（一次问完）：

```
为了建立记忆系统，需要了解几件事：
1. 这个工作/项目是关于什么的？（一句话）
2. 当前最重要的目标是什么？
3. 有什么特殊规则、踩过的坑、或必须记住的约定？
4. 你的工作偏好（风格、协作方式等）？
5. 有哪些常用的多步骤流程需要文档化？
```

**Step 3**：根据回答生成初始文件（见[文件格式](#文件格式)）。

**Step 4**：告知用户创建了哪些文件，hot.md 的 token 预估值。

---

### 蒸馏（会话结束 / 任务完成后）

触发词：「记住这次会话」「今天先到这」「帮我记录一下」「任务结束了更新记忆」

逐项检查，**有则更新，无则跳过**，最后明确告知用户处理结果：

**① 是否发现新的规则、踩坑或关键约定？**
→ 更新 `hot.md`（一句话摘要）和 `context.md`（详情与原因）

**② 用户是否纠正了某个行为或偏好？**
→ 更新 `user.md`

**③ 是否形成了可复用的多步骤流程？**
→ 在 `sop/` 创建流程文档（见 [SOP 格式](#sop-格式)），并在 `hot.md` 添加触发索引

**④ hot.md 是否超过 600 tokens（约 2400 字符）？**
→ 将较旧/较少用的条目迁移到 `context.md`，hot.md 只保留简短引用

**⑤ 本次有什么需要下次优先处理的？**
→ 写入 `guidance.md`（覆盖），追加摘要到 `log.md`

**蒸馏质量原则**（详见 `references/memory-principles.md`）：
- 具体胜于抽象：写可直接执行的规则，不写模糊概括
- 蒸馏不是搬运：提炼"下次有用的精华"，不粘贴原始对话
- 无更新时明确告知，不做无意义写入

---

### 审计（检查记忆健康度）

触发词：「记忆审计」「整理一下记忆」「检查记忆质量」

**Step 1**：读取所有记忆文件和字符统计。

**Step 2**：按维度评分（🟢 良好 / 🟡 需改进 / 🔴 有问题）：

| 维度 | 检查项 |
|------|--------|
| 热记忆精简度 | hot.md < 600 tokens？有无冗余废话？ |
| 信息密度 | 每条记忆具体可操作？有无模糊无用条目？ |
| 分层合理性 | 该在热区的是否被埋在温区？该下沉的是否还在热区？ |
| 时效性 | 有无已过时、不再适用的条目？ |
| user.md 完整性 | 用户偏好是否有记录？是否与实际一致？ |
| SOP 覆盖度 | 有无常用但未文档化的流程？ |

**Step 3**：给出具体改动建议（不只说"建议更新"，要指明改什么），询问是否立即执行。

---

### 单条更新

触发词：「记住 X」「把这个记下来」「更新一下偏好」

1. 判断信息属于哪一层（参考层级选择规则）
2. 检查是否已有相关条目（避免重复）
3. 写入，告知用户写入了哪个文件的哪个位置

## 文件格式

**`hot.md`**（热记忆，<600 tokens）
```markdown
# Hot Memory
_最后更新：{日期}_

## 核心事实
- {高频规则/约定，一句话}
- {例：测试前必须先 seed:test，否则数据缺失}

## 已知雷区
- {踩过的坑，简短}

## 扩展索引
- 详细背景 → context.md
- 用户偏好 → user.md
- 部署流程 → sop/deploy.md
```

**`context.md`**（温记忆，详细）
```markdown
# Context
_最后更新：{日期}_

## 工作概述
{一句话描述}

## 目标
- 短期：{...}  长期：{...}

## 决策历史
- {决策}：{原因/背景}（{日期}）

## 已知问题 / 注意事项
- {具体问题}（{发现时间}）

## 阶段性成果
- {成果}（{日期}）
```

**`user.md`**（用户画像，跨工作区复用）
```markdown
# User Profile
_最后更新：{日期}_

## 协作风格
- {例：重要改动先给 plan 等确认，小改动直接做}

## 偏好
- {具体偏好，例：函数式 > class；禁止未经询问引入新依赖}

## 沟通方式
- {例：解释简洁，有问题直接说不要迂回}

## 禁忌
- {明确不喜欢的行为}
```

**`guidance.md`**（主动建议，每次覆盖）
```markdown
# Guidance
_上次会话（{日期}）生成_

- {具体建议，附必要背景，例：auth.ts 类型问题还没修，下次优先处理}
```

**`log.md`**（会话日志，倒序追加）
```markdown
# Log

## {日期} — {标题}
**产出**：{核心成果}
**决策**：{若有}
**遗留**：{若有}
```

## SOP 格式

在 `sop/` 目录下创建流程文档：

```markdown
# {流程名} SOP

**适用场景**：{何时执行}
**前置条件**：{需要什么}

## 步骤

1. {Step 1}
   ```bash
   {具体命令}
   ```
   ⚠️ {容易出错的地方}

2. {Step 2}...

## 验证
- {如何确认成功}

## 已知问题
- {症状} → {解决方法}
```

在 `hot.md` 的"扩展索引"中添加明确触发条件：`执行 X 操作 → sop/x.md`

## 工具集成

**支持 skill 系统**（Claude Code、OpenCode、OpenClaw）：安装后直接触发。

**支持系统提示词**（Cursor、Windsurf 等）：在对应配置文件（`AGENTS.md` / `.cursorrules` 等）中添加：
```
记忆文件在 .memory/（hot/context/user/guidance/log.md，sop/）。
会话开始时读取 hot.md 和 guidance.md 简述要点；
用户说「记一下」或「今天先到这」时执行记忆蒸馏。
```

**其他任意 AI 工具**：对话开头粘贴：
```
帮我管理会话记忆，文件在 .memory/。
我说「看看记忆」时读取简述；说「今天先到这」时蒸馏写入。
hot.md 严控 <600 tokens；guidance.md 覆盖写入；其他文件增量追加。
```
