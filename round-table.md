# RoundTable：一人公司的多Agent圆桌决策系统

RoundTable 是基于 Claude Code subagent 机制实现的多人角色决策模拟系统，让你一个人也能拥有完整的智囊团，从不同专业角度评估想法、讨论方案、做出决策。

## 🎯 核心特点

- **真正独立的角色人格**：每个角色都是独立的 subagent，拥有自己的上下文窗口和人格设定，不会互相干扰
- **多视角碰撞**：产品、技术、市场、魔鬼代言人等不同角色从各自专业角度发言，避免思维盲区
- **结构化讨论流程**：从提案梳理 → 各方表态 → 交叉质疑 → 共识提炼，完整的讨论框架
- **组织上下文感知**：自动加载你的背景信息，让讨论更贴合你的实际情况
- **决策可追溯**：所有讨论记录自动归档，方便后续复盘

## 🚀 快速开始

### 1. 安装

```bash
cd round-table
chmod +x install.sh
./install.sh
```

安装完成后重启 Claude Code 即可使用。

### 2. 配置

编辑 `~/.roundtable/org/context.md` 填写你的个人/组织背景信息：
- 领域/行业
- 技术栈偏好
- 商业模式
- 时间投入/预算/技术能力
- 已有资源
- 决策风格/风险偏好

### 3. 开始使用

在 Claude Code 中输入：
```
圆桌讨论 你的想法/提案/问题
```

例如：
```
圆桌讨论 我想做一个AI写作工具，专注于公众号文章生成，你觉得可行吗？
```

## 📋 讨论流程

### 第1轮：议题确认
主持人复述你的议题，模糊的地方会主动澄清，同时展示本次圆桌的角色阵容。

### 第2轮：各方表态
所有角色并行独立思考，分别从自己的专业角度给出初步评价：
- 立场：支持/谨慎/反对
- 专业分析：核心判断 + 支撑理由 + 关键风险
- 核心观点一句话总结

### 第3轮：交叉质疑
找出最大的2-3个分歧点，让相关角色交锋，魔鬼代言人会专门找出所有默认成立但未经验证的假设。

### 第4轮：聚焦深挖（可选）
你可以指定方向，让相关角色深入讨论特定话题。

### 第5轮：共识提炼
主持人综合所有观点，产出最终结论：
- 共识点
- 核心分歧
- 风险排序
- 决策选项

## 📂 目录结构

```
~/.roundtable/
├── org/
│   └── context.md          # 组织/个人上下文配置
└── projects/               # 项目讨论记录归档
    └── {project_name}/
        └── round-table-{date}.md
```

## ⚙️ 自定义

### 新增角色
在 `round-table/` 目录下新建 `rt-{role}.md` 文件，按照现有角色的格式定义新角色人格，重新运行安装脚本即可。

### 修改角色人格
直接编辑 `round-table/` 目录下对应的角色文件，重新运行安装脚本更新。

## 💡 使用场景

- 产品idea可行性评估
- 技术方案选型讨论
- 创业方向分析
- 重要决策前的风险评估
- 思维体操，避免认知盲区

## 🔧 技术原理

基于 Claude Code 的 Agent 工具实现：
- 主 Agent 作为主持人，负责编排讨论流程
- 每个角色 spawn 独立的 subagent，拥有独立上下文
- 通过 SendMessage 实现角色间的对话和交叉质疑
- 所有 subagent 的输出由主持人统一整理呈现
