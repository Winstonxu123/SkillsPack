# memory-manager

跨会话分层记忆管理 skill。帮助 AI 助手在会话结束时提炼关键信息写入结构化记忆文件，在下次会话开始时读取并恢复上下文，实现持久化记忆。

适用于任何领域（编程、写作、研究、运营等）和任何支持文件读写的 AI 工具。

---

## 安装

```bash
/skill install memory-manager.skill
```

---

## 记忆架构

所有记忆文件存放在工作目录的 `.memory/` 下：

```
.memory/
├── hot.md        # 热记忆：每次必读的核心事实，严控 <600 tokens
├── context.md    # 温记忆：详细背景、决策历史、已知问题
├── user.md       # 用户画像：稳定的个人偏好，跨工作区复用
├── guidance.md   # 主动建议：给下次会话的优先提醒（每次覆盖）
├── log.md        # 会话日志：历次摘要，倒序追加
└── sop/          # 操作流程：可复用的多步骤 SOP 文档
```

核心原则：越频繁调用的记忆越要精简，越细节的内容越要下沉。

---

## 使用方式

### 会话开始时
> 「看看记忆」「回顾上次进度」

读取 `hot.md` 和 `guidance.md`，简述遗留事项和当前上下文。

### 会话结束时
> 「今天先到这」「记住这次会话」「帮我记录一下」

提炼本次会话，按维度分层写入：新规则/踩坑 → `hot.md` + `context.md`，用户偏好纠正 → `user.md`，复杂流程 → `sop/`，下次优先事项 → `guidance.md`，摘要 → `log.md`。

### 初始化
> 「初始化记忆」「这是新项目」

从零建立记忆结构，引导收集项目背景、目标、偏好、已知约定。

### 记忆审计
> 「记忆审计」「整理一下记忆」

检查各层记忆的健康状况（精简度、时效性、分层合理性），给出具体优化建议。

### 随时记录
> 「把这个记下来」「记住这个偏好」

判断信息所属层级，立即写入对应文件。

## 工具集成

**支持 skill 系统**（Claude Code、OpenCode、OpenClaw）：安装后直接用触发词激活。

**支持系统提示词**（Cursor、Windsurf 等）：在 `AGENTS.md` 或工具对应配置文件中加入：

```
记忆文件在 .memory/（hot/context/user/guidance/log.md，sop/）。
会话开始时读取 hot.md 和 guidance.md；用户说「记一下」时执行蒸馏写入。
```

**其他任意工具**：对话开头粘贴：

```
帮我管理会话记忆，文件在 .memory/。
我说「看看记忆」时读取简述；说「今天先到这」时蒸馏写入。
hot.md 严控 <600 tokens；guidance.md 覆盖写入；其他文件增量追加。
```
