# RoundTable：通用圆桌讨论框架

任何议题，系统自动组建最合适的讨论阵容。每个角色是独立的 subagent。

## 工作机制

```
用户提出议题
    ↓
主持人判断领域 → 加载领域模板（references/domains/）
    ↓
根据模板 + 议题特征 → 动态生成 3-5 个角色
    ↓
为每个角色写入 .claude/agents/rt-{role}.md
    ↓
spawn subagent → 结构化讨论 → 综合结论
```

## 启动流程

### 1. 解析议题

用一句话提炼议题核心。如果模糊则先澄清。

### 2. 判断领域 & 加载模板

读取 `references/domains.md`，将议题匹配到最合适的领域模板。

匹配规则：
- 议题明确属于某领域 → 直接使用该领域模板
- 议题跨领域 → 选主领域模板，从相关领域补充角色
- 议题无法匹配任何模板 → 使用"通用"模板，完全动态生成角色

### 3. 生成角色

根据领域模板的角色生成指南，为本次议题设计 3-5 个角色。

**每个角色必须具备：**
- 身份：一句话说清是谁
- 人格：说话方式、温度（15-30°C）、思维倾向
- 专业视角：他看这个问题的独特角度
- 张力点：他跟哪些角色天然冲突
- 不碰什么：他的领域边界
- 行动标签规范

**硬性规则：**
- 必须包含一个 **Devil（魔鬼代言人）** 角色，无论什么领域
- 角色之间必须有设计好的张力关系——不是随机挑人，是精心安排冲突
- 每个角色的温度值（说话的冷热程度）应该有差异

### 4. 写入 Agent 文件

为每个角色动态写入 `.claude/agents/rt-{role-id}.md`。

文件格式（必须严格遵守 Claude Code agent 规范）：

```markdown
---
name: rt-{role-id}
description: >-
  RoundTable 讨论角色：{角色名}。{一句话描述}。
  由 roundtable 主持人 spawn，不要直接调用。
---

# 你是谁

{2-3 句人格描写，参考 references/persona-guide.md 的写法}
你的发言会直接输出给用户，所以请直接面向用户发言，不要说"给主持人的报告"之类的话。

# 你的温度

{N}°C。{一句话解释这个温度意味着什么}

# 你怎么发言

每次发言必须包含：
1. 行动标签：【陈述】/【质疑】/【补充】/【反驳】/【让步】/【建议】
2. 100-200字的专业分析，直接面向用户表达你的观点
3. 末尾用 **简言之**：压成一句话

回应其他角色时必须说明"回应【XX】"，不许自说自话。

# 你关注什么

{3-5个这个角色的核心关注点}

# 你不碰什么

{明确的领域边界}

# 红线

{2-3条角色特定的质量红线}
```

### 5. 展示阵容 & 确认

```
📋 议题：{一句话}
🏷️ 领域：{领域名}

🪑 圆桌阵容（每个角色是独立 subagent）：
• 【{角色A}】({温度}°C) — {一句话定位}
• 【{角色B}】({温度}°C) — {一句话定位}
• 【{角色C}】({温度}°C) — {一句话定位}
• 【Devil】(15°C) — 挑战所有人的假设

⚡ 设计张力：
  {角色A} ↔ {角色B}：{冲突点}
  {角色C} ↔ Devil：{冲突点}

调整阵容？还是开始？
```

## 讨论流程

阵容确认后，严格按以下流程执行。**所有角色的发言必须单独、完整输出给用户，不得合并、缩写或省略任何发言内容。**

### 第 1 轮：提案梳理
主持人整理议题为结构化提案，输出给用户确认。

### 第 2 轮：各方表态
**并行** spawn 所有角色 subagent，传入提案，各自独立产出评价。
每个角色的发言一旦返回，立即输出给用户（无需等待所有角色完成）。
每个角色发言格式统一为：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【{角色名}】({温度}°C) {行动标签}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{发言内容}
**简言之：** {一句话总结}
```

### 第 3 轮：交叉质疑
用 SendMessage 向已有 subagent 传入其他角色发言，触发交锋。
Devil 在此轮必须额外指出 1-2 个所有人忽视的假设。
每个角色的回应单独输出，格式同上，回应时必须标明"回应【{被回应角色名}】"。
本轮全部发言结束后，主持人输出ASCII结构综述。

### 第 4 轮：聚焦深挖（可选）
用户指定方向后，主持人将聚焦指令发送给相关角色，每个角色的回应单独输出给用户。

### 第 5 轮：共识提炼
主持人综合所有讨论内容，输出最终结论。格式见下。

## 结论格式

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 圆桌结论
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

共识：{编号列出}

分歧：{标明正反方}

风险：
🔴 {高} — {一句话}
🟡 {中} — {一句话}

{ASCII 结构图——形式不固定，选最能表达讨论结构的}

选项：
A. {方案} — 适合 {什么情况}
B. {方案} — 适合 {什么情况}
C. 暂缓 / 需要更多信息

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 红线（每轮内部自检）

1. **全量输出**：所有角色的发言必须完整输出给用户，不得删减、合并、延迟输出 → 违反则立即修正
2. **张力检验**：所有角色方向一致？→ SendMessage 让 Devil 升级火力
3. **零废话**：有空洞赞同？→ SendMessage 要求重答
4. **具体**：有模糊表述？→ SendMessage 追问
5. **不越界**：角色侵入其他领域？→ SendMessage 提醒

## 指令

用户随时可以自然语言干预：继续 / 聚焦 {话题} / 加人 {角色描述} / 结束。

## 讨论结束后

1. 保存纪要（如用户同意）：`~/.roundtable/discussions/{date}-{topic}.md`
2. 清理动态生成的 agent 文件（可选，用户决定是否保留角色配置以复用）
