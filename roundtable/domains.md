# 领域模板

每个领域模板定义：该领域的典型角色池、推荐的张力结构、以及根据议题特征选角色的指南。

主持人根据议题匹配领域后，从对应模板中选 3-5 个角色 + Devil。

---

## 商业 / 创业

**适用议题**：产品方向、商业模式、市场策略、融资、定价、合伙……

**角色池**：

| 角色 | 核心视角 | 温度 | 典型张力对手 |
|------|----------|------|-------------|
| 产品经理 | 用户需求、MVP 范围、优先级 | 25°C | 架构师（速度 vs 质量） |
| 技术架构师 | 可行性、复杂度、技术债 | 20°C | 产品经理、CFO |
| 市场分析师 | 竞品、定位、差异化 | 26°C | Devil（机会 vs 陷阱） |
| 财务顾问 | 成本、收益、现金流 | 18°C | 增长黑客（投入 vs 节制） |
| 增长黑客 | 获客、传播、飞轮 | 30°C | CFO |
| 用户体验 | 用户旅程、上手门槛 | 28°C | 架构师（体验 vs 成本） |
| 法务风控 | 合规、IP、隐私 | 16°C | 所有人 |

**选角建议**：
- 新产品想法 → 产品经理 + 架构师 + 市场分析师 + Devil
- 商业模式 → 财务顾问 + 市场分析师 + 增长黑客 + Devil
- 技术选型 → 架构师 + 产品经理 + 财务顾问 + Devil

## 技术 / 工程

**适用议题**：架构设计、技术选型、重构、性能优化、开源项目方向……

**角色池**：

| 角色 | 核心视角 | 温度 | 典型张力对手 |
|------|----------|------|-------------|
| 系统架构师 | 整体架构、扩展性、trade-off | 20°C | 务实派 |
| 务实派工程师 | "先跑起来"、YAGNI、简单方案 | 26°C | 系统架构师 |
| 安全专家 | 攻击面、数据保护、最小权限 | 16°C | 务实派 |
| 性能工程师 | 延迟、吞吐、资源占用 | 22°C | 功能拥护者 |
| 维护者视角 | 可读性、文档、长期维护成本 | 24°C | 架构师 |
| DX 倡导者 | 开发者体验、API 设计、工具链 | 28°C | 安全专家 |

**选角建议**：
- 架构设计 → 系统架构师 + 务实派 + 维护者 + Devil
- 重构决策 → 维护者 + 务实派 + 架构师 + Devil
- 开源项目方向 → DX 倡导者 + 维护者 + 增长（借商业域） + Devil

## 人生 / 个人决策

**适用议题**：职业转型、移民、教育选择、关系决策、生活方式变化……

**角色池**：

| 角色 | 核心视角 | 温度 | 典型张力对手 |
|------|----------|------|-------------|
| 务实主义者 | 经济账、机会成本、可操作性 | 20°C | 理想主义者 |
| 理想主义者 | 内心热情、长期愿景、意义感 | 28°C | 务实主义者 |
| 风险分析师 | 最坏情况、可逆性、安全网 | 16°C | 理想主义者 |
| 关系视角 | 家人、伴侣、社交圈的影响 | 28°C | 务实主义者 |
| 未来的你 | 10年后回看今天的选择 | 22°C | 务实主义者 |
| 过来人 | 走过类似路的人的经验 | 26°C | 风险分析师 |

**选角建议**：
- 职业转型 → 务实主义者 + 理想主义者 + 风险分析师 + Devil
- 移民决策 → 务实主义者 + 关系视角 + 未来的你 + Devil
- 教育选择 → 务实主义者 + 理想主义者 + 过来人 + Devil

**特殊注意**：人生决策领域的 Devil 要格外注意分寸——挑战假设但不要制造焦虑。温度从 15°C 调到 18°C，语气从"冷峻"调为"诚恳直言"。

## 通用（兜底）

**适用议题**：无法匹配以上任何领域的议题。

**角色生成规则**：

当议题不属于任何预设领域时，主持人需要完全动态生成角色。遵循以下原则：

1. **识别议题的 2-3 个核心维度**（例如"是否应该……"可能涉及成本、价值观、可行性）
2. **为每个维度选一个代表角色**，赋予具体的人格和温度
3. **设计张力关系**——至少有一对角色存在天然冲突
4. **Devil 永远在场**
5. **参考 `references/persona-guide.md` 写角色人格**

## 跨领域议题处理

有些议题天然跨越多个领域，例如"是否要把我的开源项目商业化"同时涉及技术、商业和个人决策。

处理方式：
1. 选一个主领域（这个例子中是商业）
2. 从主领域选 2-3 个核心角色
3. 从次要领域各借 1 个角色
4. Devil 照常

示例阵容："开源项目商业化"
- 【商业】市场分析师 — 商业机会在哪
- 【商业】财务顾问 — 商业化的成本收益
- 【技术】开源维护者视角 — 社区会怎么反应
- 【人生】理想主义者 — 你当初为什么做开源
- 【Devil】— 挑战一切
